{{ serviceName }} (GRIS)

The Government Research Identifier Service is a service used to create a unique ID (known as a Government Research Identifier) for a research study. This unique ID will store key information about a study, and users will be able to update this information using the Government Research Identifier Service.

Important

Please note that online v1 - v4 and offline v1 - v2 were originally designed using an older version of the GOV.UK toolkit, which used the old branding. To get the best representation of how these iterations orignally looked please refer to the Miro boards found in the sidebar.

Version 4 - Creates a GRIS ID without a login

Version 4 - Creates a GRIS ID without a login

(27/05/2025)

What we created

  • A system that generates a GRIS ID without the need of a login system - the hypothesis being that having to have more login credentials is a barrier
  • A system that can view and manage a study by providing relevant details, and without a login
  • Kickout screen added when searching for a study using incorrect credentials, GRIS-111 111 111 for GRIS ID and notfound@email.com for Chief Investigator email address
  • Implemented error messages when fields are blank or don't fit criteria when searching for a study and chief investigator email address

Known issues

  • Exploration needs to be done on how viable this is as an option in regards to security of data
  • Error messaging throughout, for example when fields are left blank, and more kickout screen, 404 etc.

Version 1 - Initial design

Version 1 - Initial design

(18/07/2025)

When searching via GRIS ID if you enter 000 in the inputs, you will get 'no results found' anything else and you will see results.

When searching via study name, if you enter investigating you will get results with pagination, anything else will show 'no results found'.

What we created

  • Login system and dashboard for call centre staff
  • Ability to search via GRIS ID or study name and manage the study
  • Register a new study in the same fashion we would on the online route using v3.1 as a base

Known issues

  • From prior projects / research call centre staff sometimes like to have a 'condensed' journey, more than one question per page so this is something we would like to try and test in version 2.
